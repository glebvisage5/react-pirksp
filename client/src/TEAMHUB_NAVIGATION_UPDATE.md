# TeamHub - Обновленная структура навигации

## Обзор изменений

Навигация TeamHub была полностью переработана для улучшения пользовательского опыта. Теперь вкладки команды отображаются в левом меню, а не внутри компонента команды.

## Новая структура навигации

### Основное меню (без выбранной команды)

Когда пользователь находится в основном режиме TeamHub, в левом меню отображаются следующие вкладки:

1. **Панель управления** (Dashboard)
   - Главная страница с обзором всех команд, проектов и задач
   - Быстрый доступ к последним действиям
   
2. **Команды** (Teams)
   - Список всех команд пользователя
   - Создание новых команд
   - Поиск и фильтрация команд

3. **Дедлайны** (Deadlines)
   - Все дедлайны из всех команд в одном месте
   - Фильтрация по статусу: Все, Сегодня, Предстоящие, Просрочено, Завершено
   - Приоритеты задач

4. **Профиль** (Profile)
   - Личная информация пользователя
   - Редактирование профиля
   - Статистика: команды, проекты, выполненные задачи

5. **Настройки** (Settings)
   - Внешний вид (тема, язык)
   - Уведомления (email, push, обновления задач и проектов)
   - Конфиденциальность и безопасность

### Меню команды (при выборе команды)

Когда пользователь выбирает команду из списка, левое меню **полностью меняется** и показывает вкладки команды:

#### Кнопка "Назад"
- В верхней части меню появляется кнопка "Назад к командам"
- Позволяет вернуться к списку команд

#### Вкладки команды (для всех ролей):

1. **Обзор** (Overview)
   - Общая информация о команде
   - Недавняя активность
   - Статистика команды

2. **Проекты** (Projects)
   - Список проектов команды
   - Создание новых проектов
   - Фильтрация и поиск проектов

3. **Технические задания** (Specifications)
   - Все ТЗ команды
   - Блочный конструктор ТЗ (как в Notion)
   - Версионирование и экспорт в PDF

4. **Задачи** (Tasks)
   - Канбан-доска с задачами
   - Фильтрация по статусу и приоритету
   - Создание задач из пунктов ТЗ

5. **Участники** (Members)
   - Список участников команды
   - Система приглашений
   - Управление ролями участников

6. **Файлы** (Files)
   - Файловый менеджер команды
   - Загрузка и организация файлов
   - Привязка к проектам и задачам

#### Вкладки команды (только для Team Leader и Moderator):

7. **Роли команды** (Roles)
   - Матрица прав доступа
   - Управление ролями: Team Leader, Moderator, Member, Viewer
   - Настройка разрешений

8. **Настройки команды** (Team Settings)
   - Общие настройки команды
   - Управление командой
   - Опасная зона (удаление команды)

## Ролевая система

### Доступные роли:

1. **Team Leader** (Лидер команды)
   - Полный доступ ко всем вкладкам
   - Управление командой и участниками
   - Доступ к Roles и Team Settings

2. **Moderator** (Модератор)
   - Доступ ко всем вкладкам кроме удаления команды
   - Управление участниками
   - Доступ к Roles и Team Settings

3. **Member** (Участник)
   - Доступ к: Overview, Projects, Specs, Tasks, Members, Files
   - Может создавать и редактировать контент
   - Нет доступа к Roles и Team Settings

4. **Viewer** (Наблюдатель)
   - Доступ только на чтение
   - Может просматривать: Overview, Projects, Specs, Tasks, Members, Files

### Логика навигации

1. **Без выбранной команды**:
   - Отображается основное меню (5 вкладок)
   - Пользователь может перемещаться между Dashboard, Teams, Deadlines, Profile, Settings

2. **С выбранной командой**:
   - Левое меню заменяется на меню команды
   - Показывается кнопка "Назад к командам"
   - Вкладки фильтруются по роли пользователя
   - В header отображается название команды вместо "TeamHub"

3. **При переключении в Admin Mode**:
   - Выбранная команда сбрасывается
   - Отображается административное меню
   - 9 модулей управления TeamHub

## Преимущества новой структуры

1. **Лучшая навигация**: Все вкладки в одном месте - в левом меню
2. **Контекстная навигация**: Меню меняется в зависимости от контекста (команда выбрана/не выбрана)
3. **Визуальная ясность**: Пользователь всегда видит, где он находится
4. **Мобильная адаптация**: Меню работает одинаково хорошо на desktop и mobile
5. **Ролевой контроль**: Вкладки автоматически фильтруются по роли

## Интеграция с Admin Mode

Admin Mode остался без изменений и продолжает работать параллельно с пользовательским режимом:

- При переключении в Admin Mode выбранная команда сбрасывается
- Отображается административное меню с 9 модулями
- При возврате в User Mode пользователь возвращается к основному меню

## Мультиязычность

Все новые компоненты поддерживают:
- Английский язык (en)
- Русский язык (ru)
- Динамическое переключение через компонент Settings

## Адаптивный дизайн

- **Desktop**: Левое меню всегда видимо, sticky позиционирование
- **Mobile**: Меню скрыто, открывается через кнопку hamburger
- **Tablet**: Адаптивное поведение в зависимости от ориентации
