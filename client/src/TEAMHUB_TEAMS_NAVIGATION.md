# TeamHub - Навигация по командам

## Обзор

Реализована полноценная система навигации по командам в TeamHub с правильной структурой: **Список команд → Страница команды → Внутренние вкладки**.

## Структура навигации

### 1. Список команд (`/components/teamhub/Teams.tsx`)

**Основной экран** - точка входа в сервис TeamHub

#### Функциональность:
- **Карточки команд** с информацией:
  - Название команды
  - Описание
  - Роль пользователя (Team Leader, Moderator, Member, Viewer)
  - Количество участников
  - Количество проектов
  - Владелец команды
  
- **Действия на карточке**:
  - Клик на карточку → открытие страницы команды
  - Меню (⋮):
    - Открыть команду
    - Настройки (только для Leader/Moderator)
    - Пригласить (только для Leader)
    - Покинуть команду (для всех кроме Leader)

- **Функции управления**:
  - Поиск команд
  - Создание новой команды
  - Фильтрация по ролям

---

### 2. Страница команды (`/components/teamhub/team-views/TeamView.tsx`)

**Главный экран работы** внутри выбранной команды

#### Верхняя панель команды:
- Аватар команды с цветом роли
- Название команды
- Роль пользователя (badge)
- Описание команды
- Статистика (участники, проекты, владелец)
- Кнопка "← Мои команды" для возврата

#### Навигационные вкладки:

| Вкладка | Компонент | Доступ | Описание |
|---------|-----------|--------|----------|
| **Обзор** | `TeamOverview` | Все | Статистика и последние изменения |
| **Проекты** | `TeamProjects` | Все | Список проектов команды |
| **ТЗ** | `TeamSpecs` | Все | Технические задания |
| **Задачи** | `TeamTasks` | Все | Канбан-доска и список задач |
| **Участники** | `TeamMembers` | Все | Управление участниками |
| **Файлы** | `TeamFiles` | Все | Файловый менеджер |
| **Роли** | `TeamRoles` | Leader, Moderator | Матрица прав доступа |
| **Настройки** | `TeamSettings` | Leader, Moderator | Настройки команды |

## Компоненты вкладок

### 1. TeamOverview (`/components/teamhub/team-views/TeamOverview.tsx`)

**Обзор команды** - dashboard с основной информацией

#### Содержимое:
- **Статистика**:
  - Проекты (общее количество, новые за месяц)
  - Задачи (общее количество, завершенные)
  - Участники (количество, новые за неделю)
  - ТЗ (общее количество, активные)

- **Последние изменения**:
  - Новые задачи
  - Обновления ТЗ
  - Новые участники
  - Созданные проекты

- **Активность за неделю**:
  - Количество действий
  - Процент роста

#### Функциональность:
- Список ТЗ с информацией:
  - Название

  - Автор
  - Дата изменения
  - Версия
  - Статус (Draft, Review, Approved, Archived)

---

### 4. TeamTasks (`/components/teamhub/team-views/TeamTasks.tsx`)

**Задачи команды** - канбан и список

#### Функциональность:
- **Два режима просмотра**:
  - Канбан-доска (4 колонки: To Do, In Progress, Review, Done)
  - Список задач

- **Информация о задаче**:
  - Название и описание
  - Статус
  - Приоритет (Low, Medium, High, Urgent)
  - Исполнитель
  - Проект
  - Срок выполнения

- **Фильтры**:
  - По проекту
  - По исполнителю
  - Поиск

- Создание задачи
- Действия: Просмотр, Редактирование, Удаление

---

### 5. TeamMembers (`/components/teamhub/team-views/TeamMembers.tsx`)

**Участники команды** - управление членами команды

#### Функциональность:
- Карточки участников с информацией:
  - Имя и email
  - Роль (Team Leader, Moderator, Member, Viewer)
  - Дата присоединения
  - Назначенные задачи
  - Завершенные задачи
  - Статус (Active/Inactive)

- **Приглашение участников** (3 способа):
  - По ссылке-приглашению
  - По email с выбором роли
  - QR-код

- Фильтр по ролям
- Поиск участников
- Действия: Просмотр профиля, Изменение роли, Удаление

---

### 6. TeamFiles (`/components/teamhub/team-views/TeamFiles.tsx`)

**Файлы команды** - файловый менеджер

#### Функциональность:
- **Два режима просмотра**:
  - Сетка (grid)
  - Список (list)

- **Типы файлов** с цветными иконками:
  - Документы (PDF, DOC)
  - Изображения (PNG, JPG)
  - Код (JS, JSX, TS)
  - Архивы (ZIP, RAR)
  - Прочее

- Информация о файле:
  - Название
  - Размер
  - Кто загрузил
  - Дата загрузки
  - Привязка к проекту

- Фильтр по типам файлов
- Поиск файлов
- Загрузка файлов
- Действия: Скачать, Удалить

---

### 7. TeamRoles (`/components/teamhub/team-views/TeamRoles.tsx`)

**Роли команды** - управление правами доступа

**Доступ**: только Team Leader и Moderator

#### Содержимое:

**1. Обзор ролей** - карточки всех ролей:
- Team Leader (Лидер команды)
  - Полный контроль над всеми аспектами
  - Все права доступа
  
- Moderator (Модератор)
  - Управление проектами, задачами, ТЗ
  - Может приглашать, но не может удалять участников
  
- Member (Участник)
  - Создание и редактирование контента
  - Стандартные рабочие права
  
- Viewer (Наблюдатель)
  - Только просмотр
  - Нет прав на изменения

**2. Матрица прав доступа** - таблица с категориями:
- **Проекты**: просмотр, создание, редактирование, удаление
- **Задачи**: просмотр, создание, редактирование, удаление, назначение
- **ТЗ**: просмотр, создание, редактирование, удаление, утверждение
- **Участники**: просмотр, приглашение, удаление, управление ролями
- **Файлы**: просмотр, загрузка, удаление
- **Настройки**: просмотр, редактирование, удаление команды

---

### 8. TeamSettings (`/components/teamhub/team-views/TeamSettings.tsx`)

**Настройки команды** - конфигурация

**Доступ**: только Team Leader и Moderator

#### 4 вкладки настроек:

**1. Основные (General)**:
- Название команды
- Описание команды
- Аватар команды

**2. Уведомления (Notifications)**:
- Новый участник
- Новая задача
- Обновление задачи
- Обновление ТЗ
- Обновление проекта

**3. Приватность (Privacy)**:
- Видимость команды (Public/Private)
- Разрешить запросы на вступление
- Требовать одобрение новых участников

**4. Опасная зона (Danger Zone)**:
- Удаление команды (с подтверждением)

## Навигационная логика

### Ключевые принципы:

1. **Изоляция данных команды**:
   - Пока команда не выбрана → показываем только список команд
   - После выбора команды → показываем только её данные
   - Нет "общей кучи" данных из разных команд

2. **Контекст навигации**:
   - Пользователь видит, в какой команде он находится (header команды)
   - Всегда есть кнопка возврата к списку команд
   - Текущая вкладка подсвечена

3. **Права доступа**:
   - Некоторые вкладки показываются только Leader/Moderator
   - Действия зависят от роли пользователя
   - Viewer имеет только права просмотра

## Интеграция

### С конструктором ТЗ:
- Кнопка "Создать ТЗ" на вкладке Specifications
- Переход на SpecBuilder с контекстом команды/проекта

### С созданием задач:
- Возможность создавать задачи из пунктов ТЗ (планируется)
- Привязка задач к проектам команды

## Примеры использования

### Открытие команды:
```typescript
// Из списка команд
<Card onClick={() => handleOpenTeam(team)}>
  {/* Карточка команды */}
</Card>

// Или через меню
<DropdownMenuItem onClick={() => handleOpenTeam(team)}>
  Открыть команду
</DropdownMenuItem>
```

### Возврат к списку:
```typescript
<Button onClick={onBack}>
  <ArrowLeft /> Мои команды
</Button>
```

### Переключение вкладок:
```typescript
<Button onClick={() => setCurrentTab("projects")}>
  <FolderKanban /> Проекты
</Button>
```
