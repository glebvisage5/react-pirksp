# Admin Mode EduCRM - Документация

## Обзор

Admin Mode - это локальная административная панель внутри EduCRM для управления всеми сущностями образовательного сервиса. Это отдельный режим работы, который активируется переключателем "Пользователь | Администратор" и предоставляет доступ к расширенным инструментам управления.

## Ключевые особенности

- **Отдельный режим работы**: Admin Mode работает параллельно с пользовательским режимом
- **Переключатель режимов**: Администраторы могут мгновенно переключаться между пользовательским и административным режимами
- **11 специализированных модулей**: Полный набор инструментов для управления всеми аспектами EduCRM
- **Локальная область действия**: Admin Mode управляет только сущностями EduCRM (не влияет на глобальный Admin Center или TeamHub)

## Доступ к Admin Mode

1. Пользователь должен иметь роль администратора (`isAdmin = true`)
2. В header появляется переключатель "Режим: Пользователь | Администратор"
3. При активации Admin Mode меню navigation полностью заменяется на административное

## Модули Admin Mode

### 1. Dashboard EduCRM
**Путь**: `/components/educrm-admin/AdminModeDashboard.tsx`

**Функционал**:
- Статистика: количество групп, студентов, старост, курсов, задач
- Средний прогресс по курсам с детализацией
- График активности за неделю
- Последние действия старост/лидеров
- Система предупреждений (пустые группы, низкий прогресс, пропущенные дедлайны)

### 2. Группы
**Путь**: `/components/educrm-admin/AdminModeGroups.tsx`

**Функционал**:
- Просмотр всех учебных групп
- Создание новой группы
- Редактирование группы (название, описание, куратор, староста, список студентов)
- Удаление группы
- Фильтрация и поиск групп
- Отображение активности группы (высокая/средняя/низкая)

### 3. Студенты
**Путь**: `/components/educrm-admin/AdminModeStudents.tsx`

**Функционал**:
- Просмотр всех студентов
- Детальная информация о студенте:
  - Прогресс по курсам
  - Выполненные задачи
  - Полученные достижения
- Фильтрация по группам, курсам, статусам
- Назначение ролей студентам
- Перенос студентов между группами

### 4. Старосты
**Путь**: `/components/educrm-admin/AdminModeElders.tsx`

**Функционал**:
- Просмотр всех старост
- Назначение нового старосты
- Назначение помощника старосты (Co-elder)
- Снятие старосты с должности
- Просмотр активности старосты
- Статистика: количество созданных задач, количество управляемых студентов

### 5. Курсы
**Путь**: `/components/educrm-admin/AdminModeCourses.tsx`

**Функционал**:
- Просмотр всех курсов
- Создание нового курса
- Редактирование курса (название, описание, автор, материалы)
- Удаление курса
- Управление доступом студентов к курсу
- Просмотр среднего прогресса по курсу
- Статусы курса: Active, Draft, Archived

### 6. Дорожные карты
**Путь**: `/components/educrm-admin/AdminModeRoadmaps.tsx`

**Функционал**:
- Создание дорожной карты обучения
- Редактирование этапов roadmap
- Привязка этапов к курсам
- Настройка дедлайнов для каждого этапа
- Просмотр прогресса по этапам
- Управление зависимостями между этапами

### 7. Задачи
**Путь**: `/components/educrm-admin/AdminModeTasks.tsx`

**Функционал**:
- Просмотр всех задач EduCRM
- Создание новой задачи
- Редактирование задачи
- Удаление задачи
- Массовое назначение задач группе
- Фильтры по: группе, студенту, курсу, статусу, приоритету
- Просмотр статистики выполнения
- Статусы: Completed, In Progress, Pending

### 8. Достижения

**Путь**: `/components/educrm-admin/AdminModeFiles.tsx`

**Функционал**:

**Доступные права**:

- Управление группами
- Просмотр студентов
- Управление студентами
- Просмотр курсов
- Управление курсами
- Создание задач
- Оценка задач
- Просмотр достижений
- Управление достижениями

### 11. Настройки EduCRM
**Путь**: `/components/educrm-admin/AdminModeSettings.tsx`

**Функционал**:
- **Внешний вид**: Тема по умолчанию (Light/Dark/System)
- **Уведомления**:
  - Включение/выключение уведомлений
  - Уведомления о новых задачах

  - Уведомления о достижениях

- **Отображение**:
  - Показывать/скрывать прогресс-бары
  - Показывать/скрывать рейтинги (leaderboards)
  - Включение/выключение геймификации
- **Интеграции**:
  - Telegram-бот для уведомлений
  - Настройка токена бота
- **Файлы**:
  - Максимальный размер файла (MB)
  - Максимальное количество файлов на пользователя
- **Курсы**:
  - Автоматическое назначение курсов
  - Требовать одобрение для записи на курс

### Интеграция в App.tsx

```typescript
// Новый тип для Admin Mode views
type AdminModeView = "dashboard" | "groups" | "students" | "elders" | 
                     "courses" | "roadmaps" | "tasks" | "achievements" | 
                     "files" | "roles" | "settings";

// State для отслеживания текущего Admin Mode view
const [adminModeView, setAdminModeView] = useState<AdminModeView>("dashboard");

// Рендеринг контента в зависимости от режима
const renderDashboardContent = () => {
  if (userMode === "admin") {
    // Показать Admin Mode компоненты
    switch (adminModeView) {
      case "dashboard": return <AdminModeDashboard />;
      case "groups": return <AdminModeGroups />;
      // ... и т.д.
    }
  } else {
    // Показать обычные пользовательские компоненты
    switch (dashboardView) {
      case "home": return <DashboardHome />;
      // ... и т.д.
    }
  }
};
```

### Навигация

В зависимости от режима (`userMode`), sidebar отображает разные наборы навигационных элементов:

- **User Mode**: Стандартное меню (Dashboard, Tasks, Courses, Group, Team, Files, Profile, Settings)
- **Admin Mode**: Административное меню (Dashboard EduCRM, Groups, Students, Elders, Courses, Roadmaps, Tasks, Achievements, Files, Roles EduCRM, Settings EduCRM)

### Цветовая схема

Admin Mode использует фирменную цветовую схему EduCRM:
- Градиент: `linear-gradient(135deg, #a855f7, #4f46e5, #22c55e)`
- Основной цвет: `#4f46e5` (indigo)
- Акцентные цвета: purple (#a855f7) и green (#22c55e)

## Использование

### Для администратора:

1. Войти в систему с правами администратора
2. Перейти в EduCRM Dashboard
3. Активировать переключатель "Admin Mode" в header
4. Навигация автоматически переключится на административное меню
5. Выбрать нужный модуль для работы
6. Переключиться обратно в User Mode когда необходимо

### Для разработчика:

```typescript
// Проверка прав доступа
const { isAdmin, userMode } = useUser();

// Отображение элементов только для админов
{isAdmin && userMode === "admin" && (
  <AdminModeComponent />
)}

// Переключение режима
setUserMode(checked ? "admin" : "user");
```

## Отличия от глобального Admin Center

| Аспект | Admin Mode (EduCRM) | Admin Center (Глобальный) |
|--------|---------------------|---------------------------|
| Область действия | Только EduCRM | Вся экосистема (EduCRM + TeamHub) |
| Доступ | Локальные администраторы EduCRM | Глобальные администраторы |
| Функции | Управление учебным процессом | Управление инфраструктурой |
| Пользователи | Управление студентами и старостами | Управление всеми пользователями системы |
| Настройки | Настройки EduCRM | Глобальные настройки системы |
