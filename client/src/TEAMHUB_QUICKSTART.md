# TeamHub - Quick Start Guide

## 🚀 Быстрый старт

### Доступ к TeamHub

1. Откройте приложение InnovateCRM
2. Войдите в систему (или зарегистрируйтесь)
3. Нажмите "Выбрать сервис"
4. Выберите **TeamHub** из списка доступных сервисов
5. Вы попадёте на главную панель TeamHub Dashboard

## 📋 Основные сценарии использования

### Сценарий 1: Создание первой команды

1. Перейдите в раздел **Teams**
2. Нажмите **"Create Team"**
3. Заполните:
   - Название команды (например, "Mobile Dev Team")
   - Описание (что делает команда)
4. Нажмите **"Create"**
5. Вы автоматически станете Team Leader

### Сценарий 2: Приглашение участников

1. Перейдите в раздел **Members**
2. Нажмите **"Invite Member"**
3. Введите:
   - Email участника
   - Выберите команду
   - Назначьте роль (Team Leader / Moderator / Member / Viewer)
4. Нажмите **"Send Invitation"**
5. Отслеживайте статус во вкладке "Invitations"

### Сценарий 3: Создание проекта

1. Перейдите в раздел **Projects**
2. Нажмите **"Create Project"**
3. Заполните:
   - Название проекта
   - Описание
   - Выберите команду
   - Установите дедлайн
4. Нажмите **"Create"**

### Сценарий 4: Создание ТЗ в конструкторе

1. Перейдите в раздел **Spec Builder**
2. Введите название ТЗ
3. Выберите проект
4. Добавляйте блоки:
   - **Heading** - для заголовков разделов
   - **Text** - для описаний
   - **List** - для перечислений требований
   - **Checklist** - для списка задач
   - **Code** - для технических примеров
   - **Table** - для структурированных данных
5. Используйте кнопки ↑↓ для перемещения блоков
6. Нажмите **"Save Spec"** для сохранения
7. Нажмите **"Preview"** для просмотра

### Сценарий 5: Создание задачи из ТЗ

**Вариант A - из чеклиста:**
1. В **Spec Builder** добавьте блок **Checklist**
2. Добавьте пункты (например, "Реализовать авторизацию")
3. Нажмите иконку ☑️ рядом с пунктом
4. Задача автоматически создастся в проекте

**Вариант B - вручную:**
1. Перейдите в раздел **Tasks**
2. Нажмите **"Create Task"**
3. Заполните:
   - Название задачи
   - Описание
   - Проект
   - Исполнитель
   - Приоритет (High/Medium/Low)
   - Статус
   - Дедлайн
4. Нажмите **"Create"**

### Сценарий 6: Работа с задачами (Kanban)

1. Перейдите в **Tasks**
2. Задачи распределены по колонкам:
   - **To Do** - новые задачи
   - **In Progress** - в работе
   - **Review** - на проверке
   - **Done** - выполнено
3. Меняйте статус задачи через меню (⋮)
4. Фильтруйте и ищите задачи

### Сценарий 7: Просмотр ТЗ

1. Перейдите в **View Specs**
2. Выберите ТЗ для просмотра
3. Используйте оглавление слева для навигации
4. Нажмите **"Download PDF"** для скачивания
5. Нажмите **"Share"** для получения ссылки
6. Нажмите **"Edit"** для редактирования

### Сценарий 8: Управление версиями

1. Перейдите в **Versions**
2. Просмотрите историю всех версий ТЗ
3. Нажмите **"Show Changes"** для просмотра изменений
4. Нажмите **"Restore"** для восстановления версии
5. Используйте **"Compare"** для сравнения версий

## 🎯 Роли и права доступа

### Team Leader (Лидер команды)
✅ Полный доступ ко всем функциям  
✅ Создание/удаление проектов  
✅ Приглашение/удаление участников  
✅ Изменение ролей  
✅ Управление настройками команды  

### Moderator (Модератор)
✅ Управление задачами  
✅ Создание/редактирование ТЗ  
✅ Назначение задач  
❌ Управление участниками  
❌ Удаление команды  

### Member (Участник)
✅ Просмотр проектов  
✅ Работа с задачами  
✅ Комментирование  
❌ Создание проектов  
❌ Управление участниками  

### Viewer (Наблюдатель)
✅ Только просмотр  
❌ Редактирование  
❌ Создание задач  
❌ Комментирование  

## 🎨 Типы блоков в Spec Builder

| Блок | Назначение | Пример использования |
|------|------------|---------------------|
| **Heading** | Заголовки разделов | "Функциональные требования" |
| **Text** | Текстовые описания | Описание проекта, целей |
| **List** | Списки | Перечисление требований |
| **Table** | Таблицы | Сравнение вариантов |
| **Image** | Изображения | Схемы, скриншоты |
| **Code** | Код | API примеры, конфигурация |
| **Link** | Ссылки | Внешние ресурсы |
| **Checklist** | Чеклисты | → Автоматически в задачи |

## 🔄 Статусы

### Проекты
- **Planning** 🔵 - планирование
- **Active** 🟢 - активный
- **Completed** ✅ - завершён
- **On Hold** 🟠 - на паузе

### Задачи
- **To Do** ⚪ - к выполнению
- **In Progress** 🔵 - в работе
- **Review** 🟡 - на проверке
- **Done** 🟢 - выполнено

### Приоритеты
- **High** 🔴 - высокий
- **Medium** 🟡 - средний
- **Low** 🟢 - низкий

## 💡 Полезные советы

### Организация работы

1. **Создайте структуру команд**
   - Разделите большую команду на подкоманды
   - Назначьте модераторов для подкоманд

2. **Используйте шаблоны ТЗ**
   - Создайте базовую структуру ТЗ
   - Копируйте и адаптируйте для новых проектов

3. **Связывайте сущности**
   - Проект → ТЗ → Задачи
   - Всё в одном месте

4. **Версионирование**
   - Делайте коммиты с понятными комментариями
   - Важные изменения - новая версия

5. **Чеклисты → Задачи**
   - Описывайте требования как чеклисты
   - Конвертируйте в задачи одним кликом

### Лучшие практики

✅ **DO:**
- Давайте понятные названия командам и проектам
- Устанавливайте реалистичные дедлайны
- Регулярно обновляйте статусы задач
- Документируйте изменения в версиях ТЗ
- Назначайте конкретных исполнителей

❌ **DON'T:**
- Не создавайте слишком много команд без необходимости
- Не пропускайте описания в проектах и задачах
- Не забывайте про дедлайны
- Не игнорируйте версионирование
- Не назначайте всё на одного человека

## ❓ FAQ

**Q: Могу ли я быть в нескольких командах?**  
A: Да! Вы можете создавать и вступать в неограниченное количество команд.

**Q: Могу ли я иметь разные роли в разных командах?**  
A: Да! В одной команде вы можете быть Team Leader, а в другой - Member.

**Q: Как экспортировать ТЗ в PDF?**  
A: Откройте ТЗ в View Specs и нажмите "Download PDF".

**Q: Можно ли восстановить старую версию ТЗ?**  
A: Да! В разделе Versions выберите нужную версию и нажмите "Restore".

**Q: Как создать задачу из пункта ТЗ?**  
A: Используйте блок Checklist в Spec Builder и нажмите ☑️ рядом с пунктом.

**Q: Сколько блоков можно добавить в ТЗ?**  
A: Неограниченно! Добавляйте столько блоков, сколько нужно.

**Q: Можно ли переместить блоки в ТЗ?**  
A: Да! Используйте кнопки ↑↓ или drag & drop (будет добавлено).

**Q: Как отозвать приглашение?**  
A: В разделе Members → Invitations нажмите "Revoke" у нужного приглашения.
